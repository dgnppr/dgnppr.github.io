## 메달리언 아키텍처란 무엇인가

데이터 레이크하우스에서 데이터를 점진적으로 정제,구조화하기 위한 설계 패턴이다.
Bronze > Silver > Gold 세 계층을 거치며 데이터의 품질과 활용도가 단계적으로 높아진다.
멀티홉 아키텍처라고도 불린다.

## 왜 메달리언 아키텍처를 사용하는가

데이터 파이프라인에서 자주 생기는 문제는 두가지이다.
1. 원본을 그대로 분석에 쓰면 더럽고(결측값, 중복, 타입 불일치), 깨지기 쉽고, 추적이 안된다.
2. 처음부터 완벽하게 가공하면 나중에 요구가 바뀌었을때 다시 만들 원본이 없다.

메달리언 아키텍처는 이 둘을 계층 분리로 해결한다. 원본은 손대지 않고 보존하고 (Bronze), 정제는 그 위에서 하고(Silver), 소비용 가공은 또 그 위에서 한다(Gold)
각 단계가 명확한 책임을 가지므로 품질이 한방향으로 점진적으로 좋아지고, 문제가 생기면 어느 층에서 깨졌는지 추적하기 쉽다.

## Bronze 계층 - 원본 계층 (Raw)

> 판단하지 않고, 받은 그대로 쌓는다.

소스 시스템에서 들어온 데이터를 거의 그대로 적재하는 층이다.
- 상태: 원본 그대로, append-only(추가만), 이력 전체 보존
- 변환: 최소화 (변환하지 않는 것이 원칙)
- 추가: 적재 메타데이터(적재시각, 소스파일명,배치ID 등)
- 목적: 감사, 재처리, 원번 보존
- 핵심 Role: 나중에 정제 로직이 바뀌어도 여기서 다시 만들 수 있다.

## Silver 계층 - 정제 계층(Cleansed/Conformed)

> 더러운 원본을 믿을 수 있는 데이터로 바꾼다.

원본을 신뢰할 수 있는 상태로 만드는 층이다.
- 상태: 정제,정합된 데이터, 비교적 정규화된 형태유지
- 변환: 결측 처리, 중복 제거, 타입 확정, 표준화, 조인을 통한 엔티티 통합
- 핵심 작업: 엔티티 해소 (Entity Resolution)
  - 흩어진 식별자를 정리해 고객, 상품, 거래 같은 핵심 비즈니스 엔티티의 신뢰할 수 있는 통합 뷰를 만든다
- 자주 다르는 것: CDC 처리, SCD 이력 관리
- 목적: 조직 전반이 공유하는 정제된 단일 출처

## Gold 계층 - 소비 계층 (Curated/Buseinss-level)

> 질문에 맞춰 조립된 바로 쓰는 데이터.

특정 용도에 맞춰 소비 직전 형태로 가공한 층이다.
- 상태: 비정규화,집계된 형태, 읽기 최적화(read-optimized)
- 변환: 집계, KPI/지표 계산, 조인을 통한 와이드 테이블 구성
- 특징: 용도별로 만든다 - BI 대시보드용, 리포트용, ML피처용, 부서별 마트 등
- 목적: 분석가,대비소드,애플리케이션 이 바로 쓸 수 있는 결과물

## 비교

| 구분     | Bronze      | Silver            | Gold              |
|--------|-------------|-------------------|-------------------|
| 별칭     | Raw/Landing | Cleased/Conformed | Cureated/Business |
| 데이터 상태 | 원본 구조       | 정제,정합됨            | 집계,소비용            |
| 정규화    | 원본 구조       | 비교적 정규화           | 비정규화(와이드)         |
| 주요 작업  | 적재,메타부착     | 정제,중복 제거,엔티티 통합   | 집계,지표 계산          |
| 변환 강도  | 거의 없음       | 중간                | 높음                |
| 주 소비자  | 데이터 엔지니어    | 데이터 엔지니어,분석가      | 분석가,BI,앱,ML       |
| 품질     | 낮음(원본)      | 높음(검증됨)           | 높음(목적 최적화)        |
| 최적화 대상 | 보존,재처리      | 정합성,재사용           | 읽기 성능             |

## 데이터 흐름

```shell
소스 시스템 (DB, 로그, API, 파일)
        │  적재(ingestion)
        ▼
   ┌─────────┐
   │ Bronze  │  원본 보존 · append-only
   └────┬────┘
        │  정제 · 중복 제거 · 엔티티 통합
        ▼
   ┌─────────┐
   │ Silver  │  신뢰 가능한 단일 출처
   └────┬────┘
        │  집계 · 지표 · 용도별 가공
        ▼
   ┌─────────┐
   │  Gold   │  소비 직전 형태
   └────┬────┘
        │
        ▼
  BI · 리포트 · ML · 애플리케이션
```

의존성은 항상 단방향으로 흐른다. 위로 거슬러 올라가는 참조가 없어야 의존성 그래프가 꺠끗하게 유지된다.

## 핵심 원칙

1. 점진적 정제: 한 번에 완벽하게 만들지 않고, 계층마다 조금씩 품질을 올린다.
2. 재처리 가능성: Bronze에 원본이 살아 있으므로, 로직이 바뀌면 Silver,Gold를 언제든 다시 만들 수 있다.
3. 단일 출처(Single Source Of Truth): 정제된 진실은 Silver에 한 번만 둔다. Gold가 여러개여도 모두 같은 Silver를 본다.
4. 멱등성: 같은 입력으로 여러번 돌려도 결과가 같아야 한다. (Merge 패턴)
5. 계약으로서의 경계: 계층 사이에 데이터 품질(DQ) 검증을 두어, 나쁜 데이터가 다음 층으로 넘어가지 못하게 막는다.
6. 증분 처리: 매번 전체를 다시 만들지 않고, 바뀐 부분만 처리한다.

## 흔한 변형

> 계층수는 미덕이 아니다. 소비처의 다양성과 거버넌스 요구에 비례해서 늘린다. 단순한 파이프라인은 3계층, 떄로는 2계층으로 충분하다.

표준은 3계층이지만, 실무에서는 상황에 따라 변형한다.
- Landing/Raw 추가: Bronze 앞에 임시 적재(landing) 영역을 둬서 적재와 보존을 분리
- Gold 분할: Gold를 재사용 가능한 원자단위와 용도별 조립으로 나누기. 소비처가 많을때 지표 정의의 단일 출처를 지키기 위함.
- Platinum 계층: Gold 위에 외부 공유,제품화용 계층을 추가하기도함.
- Bronze>Gold 직결: 정제가 거의 필요없는 단순 케이스는 Silver를 건너뛰기도함.

## 장단점

장점
- 관심사 분리: 각 층의 책임이 명확해 유지보수,디버깅이 쉽다.
- 추적성: 데이터가 어디서 어떻게 변했는지 따라가기 좋다.
- 재처리 유연성: 원본이 보존되어 로직 변경에 강하다.
- 재사용성: 하나의 Silver를 여러 Gold가 공유한다.
- 품질의 점진적 향상: 단계마다 검증 지점을 둘 수 있다.

단점:
- 저장 비용
- 지연: 계층을 거칠수록 최종 데이터까지 시간이 걸림
- 복잡도,오버헤듸: 소규모/단순 파이프라인엔 과할 수 있다.
- 중복 로직 위험: Gold가 많아지면 집계 로직이 흩어져 정의가 갈라질 수 있다. (-> 단일 출처 원칙으로 방어)

## 베스트 프랙티스 & 흔한 실수

Best Practice
- Bronze에는 절대 변환을 넣지 않는다 (보존 가치를 지킨다.)
- 비즈니스 로직,집계는 Gold에 모은다(Silver는 정합성까지만)
- 계층 경계마다 DQ검증/Assertion을 둔다.
- 적재 메타데이터(배치시각,소스ID)를 입급 시민으로 관리한다.
- 변환은 멱등하게 설계한다.

흔한 실수
- Bronze에서 정제까지 해버려 원본 보존 가치를 잃는다.
- 비즈니스 로직이 Silver와 Gold에 중복되어 정의가 갈라진다.
- 계층 간 의존성이 양방향으로 얽혀 DAG가 스파게티가 된다.
- 적재 메타 설계를 빠뜨려, 백필/재적재 때 중복이 발생한다.
- 소비처가 하나뿐인데 과도하게 계층을 쪼개 오버헤드만 키운다.

## 요약

| 질문     | 답                           |
|--------|-----------------------------|
| 한 줄 정의 | 데이터를 B>S>G로 거치며 정제하는 패턴     |
| Bronze는 | 원본 보존, 변환 없음, append-only   |
| Silver는 | 정제,정합,신뢰 가능한 엔티티의 단일 출처     |
| Gold는  | 집계소비용, 용도별 읽기 최적화           |
| 핵심 가치  | 점진적 품질 향상 + 재처리 가능성 + 단일 출처 |
| 언제 과한가 | 소비처가 적고 파이프라인이 단순할때         |
