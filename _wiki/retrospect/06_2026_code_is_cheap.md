# 과거

우연찮게 오랜만에 나의 블로그에 들어와서 여러 글을 다시 읽어보았다. 
불과 2년 전에 쓴 글만 보더라도 자바, 스프링 등 내부 구현과 시스템 디자인핮 관심이 있었던 편이다. 
2년전만 하더라도 코드를 잘 작성하는 것이 개발자의 전부는 아니지만, 정말 중요한 능력이라고 생각했었다.
GPT의 등장 이후에도 코드 퀄리티나 할루시네이션 때문에 LLM을 전적으로 믿지는 않았고 보조 수단으로만 사용하였다.


<br>

# 현재

하지만 현재는 모든 것이 바뀌었고, 또 모든 것이 바뀔 것 같다. 이제는 코드 작성 비용이 0에 수렴한 것 같다.
코드 속도를 내가 못따라가는 수준이 되었고, 더이상 코드를 잘 짜는 것이 예전만큼 중요해지지 않은 것 같다.
물론 좋은 코드와 아키텍처를 짤 수 있어야 LLM이 이상하게 작성한 코드를 검증하고 수정할 수 있는 것은 당연하다.
하지만 예전처럼 개발자만이 코드를 작성하는 시대는 지났고, 누구나 자신의 생각을 프롬프트를 사용하여 코드로 옮길 수 있는 시대가 되었다.

그렇다고 하면 이제 정말 중요한 능력이 무엇일까?
내가 생각할 때 현재 가장 중요한 능력은 갑자기 도래되는 것이 아니고, 문제를 해결하는 능력이다.
이 능력은 과거나 현재나 미래나 동일할 것이다.

그렇다고 하면 문제를 어떻게 잘 해결하냐를 잘 고민해봐야한다.
