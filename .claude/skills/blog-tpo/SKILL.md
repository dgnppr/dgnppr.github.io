# Blog TPO Skill

DRAGONAPPEAR 기술 블로그 포스트의 기획과 콘텐츠 전략을 수립한다.

## 작업 순서

### 1. 블로그 현황 파악

다음 파일을 읽어 현재 콘텐츠 구조를 파악한다:
- `data/tag_count.json` — 기존 태그 및 사용 빈도
- `data/total-document-url-list.json` — 기존 포스트 전체 목록
- `_config.yml` — 블로그 기본 설정

### 2. 주제 분석

- **대상 독자 정의**: 주니어(0~2년) / 미들(3~5년) / 시니어(5년+)
- **포스트 유형 선택**: 튜토리얼 / 심층 분석 / 개념 설명 / 회고 / 비교 분석
- **중복 확인**: `total-document-url-list.json`으로 유사 주제 기존 포스트 확인
- **카테고리 배치**: `_wiki/` 하위 적절한 디렉토리 결정

### 3. `_workspace/01_tpo_brief.md` 작성

아래 구조를 반드시 따른다:

```markdown
# TPO Brief: {포스트 제목}

## 기본 정보
- 포스트 유형: 튜토리얼 / 심층 분석 / 개념 설명 / 회고
- 대상 독자: 주니어 / 미들 / 시니어
- 예상 분량: ~{N}단어
- 배치 경로: _wiki/{category}/{slug}.md

## SEO 정보
- 메타 제목: {60자 이내}
- 핵심 키워드: {3~5개}
- 추천 태그: {기존 태그 우선, tag_count.json 참조}

## 상세 아웃라인

### 도입부
- 왜 이 주제를 다루는가
- 이 글을 읽으면 무엇을 얻는가

### {H2 섹션 1}
#### {H3 포인트 1}
- 핵심 내용 요약
#### {H3 포인트 2}
- 핵심 내용 요약

### {H2 섹션 2}
...

### 마무리
- 핵심 요약
- 다음 단계 또는 관련 포스트 추천

## 기술 검증 포인트
- {불확실한 항목}: 검증 필요
```

### 4. 기술 정확성 체크

아웃라인의 기술 내용이 정확한지 확인한다. 불확실한 내용은 "검증 필요" 표시 후 Frontend에 검증을 요청한다.

## 산출물 기준

- 아웃라인은 Frontend가 실제 글을 쓸 수 있을 만큼 구체적이어야 한다
- 각 H2 섹션에 최소 2개의 H3 포인트를 포함한다
- 기술 용어는 한국어 표기와 영어 원문을 병기한다 (예: 가상 스레드(Virtual Thread))
