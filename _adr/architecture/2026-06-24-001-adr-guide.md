## Context

이 블로그에 ADR(Architecture Decision Records) 시스템을 도입했다.  
아키텍처·기술 결정의 맥락과 이유를 기록하고, Claude Code에서 바로 조회·작성할 수 있도록 MCP 서버(`dgnppr-adr`)와 연동한다.

이 문서는 ADR 시스템 자체의 사용법을 기술한다.

## Decision

`_adr/` Jekyll 컬렉션 + `dgnppr-adr` MCP 서버 + `/adr` Claude Code 스킬 조합으로 관리한다.


- `NNN`: 카테고리 내 순번 (3자리, 001부터)
- `slug`: 소문자 하이픈 구분 요약

## Claude Code에서 사용하기

Claude Code 세션에서 `/adr` 스킬로 조회·작성한다.

### 명령 목록

| 명령 | 동작 |
|------|------|
| `/adr` 또는 `/adr list` | 전체 ADR 목록 (상태별 그룹) |
| `/adr search <키워드>` | 제목 + 본문 키워드 검색 |
| `/adr find <자연어>` | 임베딩 유사도 검색 — 관련 ADR 목록 반환 |
| `/adr query <질문>` | 임베딩 검색 후 관련 ADR 본문 기반 답변 |
| `/adr read <경로>` | 특정 ADR 읽기 |
| `/adr write <경로>` | ADR 작성 또는 수정 |

### 예시

```
/adr list
/adr search kafka
/adr find "메시지 큐 선택 이유"
/adr query "왜 PostgreSQL을 선택했나요?"
/adr read architecture/2026-001-use-kafka.md
/adr write architecture/2026-003-adopt-redis.md
```

### 자연어로도 가능

```
ADR 목록 보여줘
kafka 관련 ADR 찾아줘
데이터베이스 선택 관련 ADR 작성해줘
```

## MCP 도구 직접 사용

`dgnppr-adr` MCP 서버 도구를 직접 호출할 수도 있다.

| 도구 | 설명 |
|------|------|
| `adr_list` | 전체 목록 반환 |
| `adr_search` | 키워드 검색 |
| `adr_find` | 임베딩 유사도 검색 (목록) |
| `adr_query` | 임베딩 검색 후 본문 반환 (QA용) |
| `adr_read` | 특정 파일 읽기 |
| `adr_write` | 파일 쓰기 |

`adr_find` / `adr_query`는 `EMBEDDING_BACKEND` 환경변수가 필요하다 (`vertexai` 또는 `ollama`).

## URL 접근

ADR은 사이트 검색·사이드바·네비게이션에 노출되지 않는다.  
직접 URL로만 접근 가능하다.

```
/adr/                          # 목록 페이지
/adr/architecture/             # 카테고리별 목록
/adr/architecture/2026-001-adr-guide/  # 개별 ADR
```

헤더 검색창의 **ADR 바로가기** 버튼으로도 목록 페이지에 진입할 수 있다.

## Consequences

- 아키텍처 결정의 맥락과 이유가 문서로 남는다
- Claude Code에서 과거 결정을 자연어로 검색·참조할 수 있다
- 블로그 사이트 검색 인덱스에는 포함되지 않아 공개 노출 최소화

## Alternatives Considered

1. **Notion 연동**: 외부 서비스 의존성 생김 — 미채택
2. **`_wiki/` 통합**: 검색 인덱스에 노출됨 — 미채택
3. **GitHub Wiki**: 블로그와 분리됨, MCP 연동 불가 — 미채택
